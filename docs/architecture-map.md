# GTG Architecture Map (Current State)

This is the Phase 7A-1 current-state architecture map for apps, packages, database, and edge functions.

## Monorepo Topology

### Applications (`apps/*`)

- `@gtg/storefront` (`apps/storefront`)
  - Public buyer experience.
  - Uses `@gtg/api` + `@gtg/config`.
- `@gtg/consultant` (`apps/consultant`)
  - Consultant dashboards and self-service flows.
  - Uses `@gtg/api` + `@gtg/config`.
- `@gtg/admin` (`apps/admin`)
  - Admin operations and back-office tools.
  - Uses `@gtg/api` + `@gtg/config`.
- `@gtg/mobile` (`apps/mobile`)
  - Placeholder workspace package (not fully scaffolded).

### Packages (`packages/*`)

- `@gtg/api`: typed wrappers for Supabase edge functions.
- `@gtg/config`: typed env/config initialization and access.
- `@gtg/supabase`: client + generated types integration.
- `@gtg/types`: shared type primitives.
- `@gtg/domain`: domain helpers/models.
- `@gtg/utils`: shared utilities.
- `@gtg/ui`: shared UI primitives.
- `@gtg/stripe`: Stripe helpers.

### Platform Layer (`supabase/*`)

- `supabase/functions`: 52 deployed edge functions.
- `supabase/migrations`: schema, RLS, constraints, and SQL RPC evolution.
- `supabase/seed.sql`: local seeding.

## Runtime Flow (High-Level)

1. App calls typed wrapper in `@gtg/api`.
2. Wrapper invokes Supabase edge function via `functions.invoke(...)`.
3. Edge function applies auth/role checks (`verifyRole`, `extractRole`) when required.
4. Edge function accesses Postgres tables and SQL RPCs through Supabase admin/user clients.
5. Function returns typed JSON payload consumed by app UI.
