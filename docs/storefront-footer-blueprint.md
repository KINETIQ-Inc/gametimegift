# GTG Footer Blueprint (Phase 1 + Phase 2)

Goal: create a footer that supports GTG conversion and trust for sports-focused decor products (vases, frames, wine racks, collectibles), while remaining visually distinct from generic ecommerce footers.

## Phase 1: Strategy + Information Architecture

### Primary objectives

- Guide high-intent shoppers to core product families quickly.
- Reinforce authenticity and trust (serial verification, licensing clarity, secure checkout).
- Support customer service and post-purchase tasks (track order, returns, shipping).
- Support B2B growth (custom/volume gifting).

### Audience paths

- Gift buyer: needs fast category discovery and delivery confidence.
- Collector: wants authenticity, licensing, and limited-drop context.
- Corporate/bulk buyer: needs quote/request path and volume ordering details.
- Existing customer: needs order tracking, returns, support.

## Phase 2: UX + Visual Blueprint

### Visual direction

- Distinct GTG identity: deep navy base, gold accent lines, cool gray content cards.
- Avoid flat generic list-only footer. Use grouped cards and a trust strip.
- Strong hierarchy:
  - headline labels in uppercase micro-heading style
  - short descriptive microcopy under key links where useful
  - newsletter module with clear value proposition

### Layout model

Desktop

- 4-column content grid + right utility rail.
- Dedicated trust strip above legal row with 3-4 trust chips:
  - Officially Licensed
  - Authenticity Verified
  - Secure Checkout
  - Easy Returns

Tablet

- 2-column grid + utility rail stacked beneath.

Mobile

- Accordion sections for each column group.
- Newsletter and social pinned near top of footer block.

### Interaction requirements

- Footer links must have visible hover/focus states.
- Newsletter CTA and verify serial link should be high contrast.
- Social icons should be keyboard focusable with labels.

### Copy guardrails

- Keep language product-specific, not generic retail jargon.
- Reinforce GTG product categories directly (vases, frames, wine racks).
- Keep legal/operational links concise and scannable.

### Success criteria for implementation phase

- Users can reach core product family pages from footer in one click.
- Verify Serial is always accessible in footer trust section.
- Footer remains readable and navigable on mobile without crowding.
- No dead links/placeholders in production map.
